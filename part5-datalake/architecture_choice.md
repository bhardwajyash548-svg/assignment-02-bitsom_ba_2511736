## Architecture Recommendation

I recommend using a Data Lakehouse architecture for the food delivery startup.

First, the system needs to handle multiple types of data such as GPS logs (semi-structured), customer reviews (unstructured text), payment transactions (structured), and images (unstructured). A data lakehouse allows storing all these data types in one system.

Second, it combines the flexibility of a data lake with the performance and structure of a data warehouse. This allows both raw data storage and efficient querying for analytics.

Third, it supports both batch and real-time processing, which is important for tracking delivery operations and analyzing customer behavior.

Therefore, a data lakehouse is the best choice as it provides scalability, flexibility, and analytical performance in a single architecture.
