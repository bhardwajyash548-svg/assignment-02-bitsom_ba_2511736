# assignment-02-bitsom_ba_2511736


## Overview

This project demonstrates end-to-end data engineering concepts, including relational databases, NoSQL systems, data warehousing, vector databases, and data lakes. The assignment is divided into six parts, each focusing on a different aspect of modern data architecture.

## Part 1 — Relational Databases (RDBMS)

- Analyzed anomalies in a denormalized dataset
- Designed normalized schema (3NF)
- Implemented SQL queries for business insights

---

## Part 2 — NoSQL (MongoDB)

- Created flexible JSON-based document models
- Performed CRUD operations using MongoDB queries
- Compared RDBMS vs NoSQL systems

---

## Part 3 — Data Warehouse

- Designed a Star Schema
- Built fact and dimension tables
- Performed analytical queries for reporting
- Applied ETL transformations on raw data

---

## Part 4 — Vector Database (AI)

- Generated embeddings using Sentence Transformers
- Built cosine similarity matrix
- Implemented semantic search functionality

---

## Part 5 — Data Lake (DuckDB)

- Queried data directly from CSV, JSON, and Parquet files
- Performed joins across multiple file formats
- Demonstrated schema-on-read approach

---

## Part 6 — Capstone Architecture

- Designed a complete hospital data system
- Integrated OLTP, OLAP, Data Lake, and Vector DB
- Built an AI-ready architecture for real-world use cases
